# 👀프로젝트 개요

정보 검색의 주도권이 기존 포털 검색에서 ChatGPT, Gemini 등 '생성형 AI'로 급격히 이동하고 있습니다. 본 프로젝트는 우리 브랜드가 AI의 답변에서 제외되거나 부정적으로 인용되는 리스크를 방지하고, AI가 선호하는 구조화 데이터(JSON 등)를 자동 생성하여 AI 검색 내 브랜드 점유율(SoM, Share of Model)을 극대화하는 SaaS 기반 지능형 플랫폼 개발을 목적으로 합니다.

> URL 입력 → 사이트 분석 → 문제점 진단 → 개선안 제시 → Before-After 비교 → 질문 패턴별 AI 응답 검증
> 

# 🖼️추진 배경 및 개발 내용

## **추진 배경**

구글 및 네이버의 검색창에 직접 키워드를 나열하는 방식인 SEO(검색 엔진 최적화) 시대가 저물고, AI가 맥락을 이해하고 정답을 제시하는 GEO(생성형 엔진 최적화) 시대가 도래했습니다. 이러한 변화의 바람에 의해 AI의 추천 리스트에 포함되지 못하는 브랜드는 디지털 시장에서 잠재적 소외 계층으로 전락할 위기에 처해 있습니다.

본 프로젝트는 이러한 시대적 요구에 발맞춰, 기업의 정보를 AI가 선호하는 구조화 데이터로 자동 최적화하여 AI 검색 노출 점유율인 SoM(Share of Model)을 극대화하는 솔루션을 개발하고자 합니다.

## **핵심 개발 내용**

- **자동화된 데이터 정제 엔진**

AI 엔진은 일반적인 서술형 문장보다 숫자, 통계, 명확한 정의가 포함된 데이터를 신뢰하며, 특히 JSON-LD와 같은 구조화된 코드를 통해 정보의 권위를 파악하는 경향이 있습니다.

따라서, 고객 웹사이트 URL 입력 시, HTML 파서를 통해 기존 데이터를 추출하고 AI 친화적인 JSON-LD 구조화 코드로 자동 변환합니다.

- **AI 맞춤형 콘텐츠 생성**

AI 엔진(Perplexity, ChatGPT 등)이 신뢰하는 통계 데이터, Q&A 형식, 비교표(Table) 데이터를 생성합니다. 브랜드 권위를 높이고 경쟁성을 더욱 높게 가져갈 수 있는 핵심 결과물입니다.

- **SaaS형 워크플로우 구축**

실시간 분석 결과물(JSONB)을 데이터베이스에 보관하고, 클라이언트가 즉시 웹사이트 메타데이터에 삽입할 수 있도록 안내하는 인터페이스를 제공합니다.

# 🤩기대 및 활용 방안

## **기대 효과**

AI 답변 내 브랜드 노출도를 최대 40% 이상 향상시키며, 검색 유입 및 구매 전환율을 극대화합니다. Adobe Analytics의 발표에 의하면, 이번 프라임데이 기간 동안 생성형 AI 기반 콘텐츠가 구매 전환에 큰 영향을 미치며, 고객 유입이 무려 3,300%나 증가했다고 합니다.

정제된 긍정 데이터와 팩트를 우선적으로 학습/인용하게 유도하여 AI의 '할루시네이션(환각)'이나 부정적인 정보 확산을 사전에 차단합니다.

- 수동 작업 없는 자동화된 JSON-LD 생성으로 정확한 정보를 AI에게 효율적이게 전달합니다. 이로써 정보의 불균형을 해결하는 강력한 마케팅 효율을 얻을 수 있습니다.

## **활용 방안**

- 엔터프라이징 마케팅 툴로 활용될 수 있습니다. 이를 통해 대규모 브랜드 웹사이트의 GEO 세팅을 자동화할 수 있습니다.
- 데이터 기반 마케팅이 가능해집니다. AI 엔진별 답변 점유율(SoM)을 대시보드로 시각화하여 마케팅 인사이트 제공.

# 프로덕트 방식

- 무단 컨설팅
    
    적당한 브랜드를 찾아서 우리의 서비스를 이용하는 가상의 고객이라고 가정한다. 해당 브랜드의 제품을 ai 수치화하고, 문제점과 json-ld 코드를 작성한다. 컨설팅 후 예상 결과를 발표한다.
    
- 가상 기업에 대한 GEO 특징 변화 잡기
    
    가상의 브랜드 웹사이트를 만들어 해당 브랜드의 장단점 및 특징을 AI가 어떻게 판단하는지 확인한다. GEO 개선 처리를 마친 이후, AI의 판단 및 검색결과가 어떻게 변동됐는지를 확인한다.

# 🌊서비스 흐름

![1000030989.jpg](attachment:c1b14fbd-8078-4b02-81ad-8b16cb02b5bd:1000030989.jpg)

![image.png](attachment:c81c4f26-89f4-421a-bd73-9fc9ca7ead8a:image.png)

1. 웹에서 고객이 GEO 주문(진단 받을 사이트의 URL 등)을 입력한다.
2. 웹 서버는 입력 받은 정보들을 저장하고, 해당 URL의 크롤링을 통해 웹페이지 구조를 추출한다. 
    
    > 정적 페이지(html) 및 동적 페이지(React, Vue)의 `<script>` 속 **ld+json와 html 본문 내용을** 추출한다.
    > 
3. AI 서버는 추출한 정보들을 토대로 GEO 개선 요소를 짚고, 그에 따른 점수를 측정한다.
    - AI가 더 잘 검색하는 조건
        
        Schema 준수 json-ld 작성 가이드 라인 : https://schema.org/docs/developers.html
        
        ‣ 
        
        https://www.tosspayments.com/blog/articles/39461
        
4. GEO 개선 결과(조언, JsonLd)를 웹 서버가 받아 DB에 보관하고, 프론트 웹에서 시각화 결과를 표시할 수 있도록 한다. 
    - 오육각형 그래프, 정수 점수, 개선안 설명 등
